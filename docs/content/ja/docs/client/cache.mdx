---
title: リクエストスコープキャッシュ
description: scope() によるリクエスト単位のキャッシュで RU 消費を削減
---

Cosmio は `scope()` によるオプトインの読み取りキャッシュを提供します。キャッシュは**デフォルトでは無効**です。リクエストごとにスコープ付きインスタンスを明示的に作成します。

## なぜデフォルトで無効なのか？

Cosmos DB のポイントリードは既に高速です（1KB あたり 1 RU）。グローバルキャッシュはリクエスト間でデータの不整合リスクがあります。`scope()` パターンはキャッシュの寿命を単一のリクエストや関数呼び出しに紐づけ、古いデータの読み取りを防ぎます。

## 基本的な使い方

```ts
const users = client.model(UserModel);

async function handleRequest(userId: string, tenantId: string) {
  // リクエストスコープのキャッシュ付きインスタンスを作成
  const scoped = users.scope();

  const user = await scoped.findById(userId, [tenantId]); // Cosmos DB から読み取り
  // ... ビジネスロジック ...
  const same = await scoped.findById(userId, [tenantId]); // キャッシュから — 0 RU

  // 関数が返ると scope は GC される → キャッシュ消滅
}
```

## Azure Functions での例

```ts
import { CosmioClient } from "cosmio";

const client = new CosmioClient({
  cosmos: { connectionString: process.env.COSMOS_CONNECTION! },
  database: "mydb",
});

export async function httpTrigger(req: HttpRequest): Promise<HttpResponseInit> {
  // 各呼び出しが独自のキャッシュを取得
  const users = client.model(UserModel).scope();
  const orders = client.model(OrderModel).scope();

  const user = await users.findById(req.params.userId, [req.params.tenantId]);
  // ... 同じリクエスト内の後続処理 ...
  const sameUser = await users.findById(req.params.userId, [req.params.tenantId]);
  // ↑ 0 RU — キャッシュから提供

  return { status: 200, jsonBody: user };
}
```

## キャッシュ設定

内部の `ReadCache` は TTL とサイズ制限をサポートしています:

- **デフォルト TTL: Infinity**（有効期限なし — スコープが GC されるまで生存）
- デフォルト最大エントリ数: 1000

リクエストスコープ（`scope()` / `withCosmioContext`）ではリクエスト終了時にキャッシュが破棄されるため、Infinity が適切です。カスタム TTL が必要な場合:

```ts
import { ReadCache } from "cosmio";

const cache = new ReadCache({ ttlMs: 5000, maxSize: 500 });
```

## ディープコピーの安全性

キャッシュされた値は**読み取りと書き込みの両方**で `structuredClone()` によりディープコピーされます。これによりキャッシュデータの変更が他の読み取りに影響しません:

```ts
const user = await scoped.findById("u1", ["acme"]);
user.name = "Modified"; // キャッシュは汚染されない
const fresh = await scoped.findById("u1", ["acme"]);
// fresh.name は元の値のまま
```

## キャッシュの無効化

全ての書き込み操作は、同じスコープ内のキャッシュを自動的に無効化します:

| 操作 | 無効化される対象 |
|------|----------------|
| `create()` | コンテナの全クエリキャッシュ |
| `upsert()` | 該当 id のポイントリードキャッシュ + 全クエリキャッシュ |
| `replace()` | 該当 id のポイントリードキャッシュ + 全クエリキャッシュ |
| `delete()` | 該当 id のポイントリードキャッシュ + 全クエリキャッシュ |
| `patch()` | 該当 id のポイントリードキャッシュ + 全クエリキャッシュ |
| `bulk()` | upsert/delete 対象のポイントリードキャッシュ + 全クエリキャッシュ |

```ts
const scoped = users.scope();
const user = await scoped.findById("u1", ["acme"]);   // キャッシュに格納
await scoped.upsert({ ...user, name: "Updated" });     // キャッシュを無効化
const fresh = await scoped.findById("u1", ["acme"]);   // DB から読み取り
```

### クエリキャッシュ

クエリ結果（`find().exec()`）もスコープ内でキャッシュされます。キャッシュキーは SQL クエリ全体・パラメータ・パーティションキーを含みます:

```
query::<container>::<sql>::<params>::<partitionKey>
```

書き込み操作は該当コンテナの全クエリキャッシュを無効化します（書き込みがクエリ結果に影響する可能性があるため）。

## 使いどころ

- **Azure Functions / サーバーレス** — 単一呼び出し内の繰り返し読み取りで RU を削減
- **API リクエストハンドラー** — ミドルウェアチェーンでの重複読み取りを回避
- **バッチ処理** — イテレーション間で参照データをキャッシュ
