---
title: Select / プロジェクション
description: 必要なフィールドのみを選択して RU 消費を削減
---

`select()` メソッドを使うと、クエリから返すフィールドを指定できます。`SELECT * FROM c` の代わりに Cosmos DB のプロジェクション（`SELECT c.field1, c.field2 FROM c`）を生成し、転送データ量を削減して RU 消費を抑えます。

```ts
const users = client.model(UserModel);
```

## 基本的な使い方

```ts
const names = await users
  .find(["acme"])
  .select("id", "name")
  .exec();
// → { id: string; name: string }[]
```

指定したフィールドのみが返されます。戻り値の型は自動的に `Pick<T, K>` に絞り込まれます。

## 型の絞り込み

`exec()` の戻り値の型は、選択されたフィールドのみを含むように絞り込まれます:

```ts
const results = await users
  .find(["acme"])
  .select("id", "name", "email")
  .exec();

// TypeScript が型を認識:
results[0].id;    // ✅ string
results[0].name;  // ✅ string
results[0].email; // ✅ string
results[0].role;  // ❌ 型エラー — 選択されていない
```

これは `ProjectedQueryBuilder<TSchema, TPaths, K>` インターフェースによって実現されており、`exec()` の戻り値の型を `DocumentRead<TSchema>[]` から `Pick<z.infer<TSchema>, K>[]` に変更します。

## 他の句との組み合わせ

`select()` はすべてのクエリビルダーメソッドと組み合わせて使えます:

```ts
const result = await users
  .find(["acme"])
  .select("id", "name", "createdAt")
  .where({ role: "admin" })
  .orderBy("createdAt", "DESC")
  .limit(10)
  .exec();
```

## RU 削減効果

Cosmos DB はドキュメントサイズに基づいて RU を課金します。必要なフィールドのみを選択することで、転送データ量が減り、RU が節約されます:

```ts
// フルドキュメント読み取り — RU が高い
const full = await users.find(["acme"]).exec();

// プロジェクション読み取り — RU が低い
const projected = await users.find(["acme"]).select("id", "name").exec();
```

これは、ドキュメントに大きなフィールド（埋め込み配列、blob、リッチテキストなど）が含まれていて、特定のクエリではそれらが不要な場合に特に効果的です。

## 生成される SQL

`select()` は `SELECT *` ではなく、対象を絞った SQL を生成します:

```ts
const query = users
  .find(["acme"])
  .select("id", "name")
  .where({ role: "admin" })
  .toQuerySpec();

// query.query → "SELECT c.id, c.name FROM c WHERE c.role = @p0"
```

デバッグには `toQuerySpec()` を使って生成された SQL を確認できます。
