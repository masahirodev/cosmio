---
title: Global Migration Registry
description: Centralized schema migration with version tracking and scope filtering
---

`MigrationRegistry` manages schema migrations across all models. Migrations are applied automatically on every read.

## Setup

```ts
import { MigrationRegistry, CosmioClient } from "cosmio";

const migrations = new MigrationRegistry({ versionField: "_v" });

migrations.register({
  name: "v2-add-fullName",
  version: 2,
  up: (doc) => {
    if (doc.firstName && !doc.fullName) {
      doc.fullName = `${doc.firstName} ${doc.lastName}`;
      delete doc.firstName;
      delete doc.lastName;
    }
    return doc;
  },
});

migrations.register({
  name: "v3-default-role",
  version: 3,
  up: (doc) => {
    if (!doc.role) doc.role = "member";
    return doc;
  },
});

// Pass to CosmioClient — all reads auto-migrate
const client = new CosmioClient({
  cosmos: { endpoint: "...", key: "..." },
  database: "mydb",
  migrations,
});
```

## Version Tracking

Each document's schema version is tracked via a version field (default: `_schemaVersion`, configurable). Documents with a version >= a migration's version skip that migration:

```ts
// Document with _v: 2 skips v2 migration but runs v3
const doc = { _v: 2, fullName: "Alice Smith" };
// After migration: { _v: 3, fullName: "Alice Smith", role: "member" }
```

## Scope Filtering

Target migrations to specific containers or models:

```ts
migrations.register({
  name: "v4-users-add-avatar",
  version: 4,
  scope: {
    containers: ["users"],     // only apply to the "users" container
    models: ["User"],          // only apply to the "User" model
  },
  up: (doc) => {
    if (!doc.avatarUrl) doc.avatarUrl = "/default-avatar.png";
    return doc;
  },
});
```

## Properties

```ts
migrations.versionField;    // "_v"
migrations.currentVersion;  // 4 (highest registered version)
migrations.migrations;      // readonly Migration[] (sorted by version)
```

## Migration Interface

```ts
interface Migration {
  name: string;           // unique name
  version: number;        // ascending version number
  scope?: {
    containers?: string[];
    models?: string[];
  };
  up: (doc: Record<string, unknown>) => Record<string, unknown>;
}
```

## Duplicate Version Protection

Registering two migrations with the same version throws an error:

```ts
migrations.register({ name: "a", version: 2, up: (d) => d });
migrations.register({ name: "b", version: 2, up: (d) => d }); // throws Error
```
